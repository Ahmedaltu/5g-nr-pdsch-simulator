BLOCK 01 / 10
LDPC Encoding
Low-Density Parity-Check codes protect your data bits by adding calculated redundancy. 5G NR uses LDPC for all data channels (PDSCH) because it parallelises efficiently in hardware.
Key idea: LDPC adds parity bits so the receiver can detect and correct errors even when some bits are flipped by noise. The parity check matrix H is sparse — mostly zeros — which is what makes decoding fast on silicon.

Why LDPC over Turbo codes?

LTE used turbo codes which decode sequentially — output of one decoder feeds the next. This caps throughput around 1 Gbps.

LDPC decodes via belief propagation on a sparse graph — all nodes update in parallel. Qualcomm implements thousands of check node processors running simultaneously, enabling 5G's multi-Gbps targets.

Code rate

Rate = k / n where k = data bits, n = codeword length.

Rate 1/2 → half the transmitted bits are parity. Strong protection, lower throughput. Rate 8/9 → very little redundancy, very fast. 5G NR supports 1/3 to 8/9 and adapts based on channel quality.

MATLAB ldpc_encode.m
% 5G NR LDPC Encoding using MATLAB 5G Toolbox
function codeword = ldpc_encode(bits, bgn, rate)
    % bgn = base graph number (1 for large TB, 2 for small TB)
    % rate = code rate (e.g. 0.5, 0.33, 0.67)
    codeword = nrLDPCEncode(bits, bgn, rate);
end

% Example usage
K        = 1000;
txBits   = randi([0 1], K, 1);
txCRC    = nrCRCEncode(txBits, '24A');   % attach CRC
codeword = nrLDPCEncode(txCRC, 1, 0.5);  % BG1, rate 1/2
Interview answer

Q: Why does 5G NR use LDPC instead of turbo codes?
Turbo codes decode sequentially with a throughput ceiling near 1 Gbps. LDPC belief propagation operates on a sparse graph where variable nodes and check nodes update in parallel — Qualcomm implements this in silicon at thousands of operations per cycle, enabling the multi-Gbps throughput 5G NR requires.

BLOCK 02 / 10
Modulation
Modulation maps bits onto complex symbols — points in a 2D constellation. Higher order modulation packs more bits per symbol but requires better SNR.
Key idea: Each constellation point encodes multiple bits. QPSK encodes 2 bits per symbol. 16QAM encodes 4. 5G NR switches modulation order adaptively based on your channel quality — this is Adaptive Modulation and Coding (AMC).

Constellation diagram — add noise to see decision errors

SNR (dB) 15 dB
● ideal points · noisy received

Gray coding

Adjacent constellation points differ by only 1 bit. When noise pushes a received signal to a neighboring wrong point, you get only 1 bit error instead of multiple. 5G NR requires Gray coding on all constellations.

Soft LLRs

The demodulator outputs Log-Likelihood Ratios — not hard 0/1 decisions. LLR >> 0 means bit is very likely 0. LLR << 0 means likely 1. LLR ≈ 0 means uncertain. The LDPC decoder uses these soft values for much better performance.

MATLAB modulate.m
function symbols = modulate(bits, M)
    % M = modulation order: 4=QPSK, 16=16QAM, 64=64QAM
    symbols = qammod(bits, M, 'InputType',       'bit', ...
                              'UnitAveragePower', true);
end

function llrs = demodulate(rxSymbols, M, noiseVar)
    % Returns soft LLRs for LDPC decoder
    llrs = qamdemod(rxSymbols, M, ...
        'OutputType',      'llr', ...
        'UnitAveragePower', true, ...
        'NoiseVariance',    noiseVar);
end
Interview answer

Q: Why does 5G switch modulation order dynamically?
Higher order modulation packs more bits per symbol but requires higher SNR to distinguish closely spaced constellation points reliably. AMC lets the base station always operate near the Shannon limit — maximising throughput for users close to the tower while maintaining reliability at the cell edge.

BLOCK 03 / 10
OFDM & 5G Numerology
OFDM splits your signal across many narrow subcarriers to defeat multipath fading. 5G NR adds flexible numerology — subcarrier spacing scales by powers of 2 to serve different bands and use cases.
Key idea: Multipath causes frequency-selective fading — some frequencies are boosted, others cancelled. OFDM makes each subcarrier so narrow it sees a flat channel, converting a hard wideband problem into many easy narrowband ones.

Resource grid — subcarriers × OFDM symbols

■ data RE ■ pilot RE ■ cyclic prefix

Cyclic prefix

The last Ncp samples of each OFDM symbol are copied to the front before transmission. As long as multipath delay spread < CP duration, intersymbol interference is completely eliminated. The receiver simply discards the CP before the FFT.

Why flexible numerology?

mmWave bands (FR2) have large phase noise that corrupts narrow subcarriers — wider spacing needed. Ultra-low latency needs shorter slots — wider subcarriers give shorter symbols. One standard serves all bands by scaling subcarrier spacing by 2^μ.

MATLAB ofdm_modulate.m
function txSig = ofdm_modulate(grid, Nfft, Ncp)
    txSig = [];
    for sym = 1:size(grid, 2)
        fd = zeros(Nfft, 1);
        fd(1:size(grid,1)) = grid(:, sym);
        td = ifft(ifftshift(fd)) * sqrt(Nfft);
        cp = td(end-Ncp+1:end);   % copy last Ncp samples
        txSig = [txSig; cp; td];  % prepend CP
    end
end
Interview answer

Q: Why does 5G NR use multiple numerologies?
Different use cases need different subcarrier spacings. mmWave bands have large phase noise requiring wider subcarriers. Ultra-low latency applications need shorter slot durations achieved with wider spacing. Sub-1GHz coverage needs narrow subcarriers for efficiency. One fixed numerology cannot serve all these cases — flexible numerology lets the same standard scale across all bands.

BLOCK 04 / 10
AWGN Channel
Additive White Gaussian Noise models thermal noise — the fundamental noise floor of every receiver. It's the baseline channel before we add multipath fading.
Key idea: rx = tx + noise. Noise is complex Gaussian with variance σ² = 1/(2·SNR_linear). AWGN is our validation baseline — if your BLER curve doesn't match theory here, something is broken in your TX or RX chain.

Signal + noise visualiser

SNR (dB) 10 dB

SNR formula

σ² = 1 / (2 × SNR_linear)

The factor of 2 is because complex noise has real and imaginary components — each with variance σ²/2. Always normalise tx signal power to 1 before adding noise.

Es/N0 vs Eb/N0

Es/N0 = energy per symbol / noise density. Eb/N0 = energy per bit. Es/N0 = Eb/N0 + 10·log10(bits/symbol). Use Es/N0 consistently throughout the simulator — mixing them shifts your BLER curve and makes results incomparable.

MATLAB awgn_channel.m
function rxSig = awgn_channel(txSig, SNR_dB)
    % Normalise tx power to 1
    txSig     = txSig / sqrt(mean(abs(txSig).^2));
    % Noise variance
    SNR_lin   = 10^(SNR_dB/10);
    sigma2    = 1 / (2 * SNR_lin);
    % Complex Gaussian noise
    noise     = sqrt(sigma2) * (randn(size(txSig)) + ...
                               1j*randn(size(txSig)));
    rxSig     = txSig + noise;
end
Interview answer

Q: Why is AWGN not sufficient to model a real 5G channel?
AWGN only models thermal noise and assumes the channel is flat and constant. Real 5G channels have multipath causing frequency-selective fading, Doppler shifts from mobility, and spatially correlated fading in massive MIMO. 3GPP defines TDL-A, TDL-C, and CDL channel models for system evaluation precisely because AWGN alone cannot capture these effects.

BLOCK 05 / 10
LS Channel Estimation
Least Squares estimation uses known pilot symbols to estimate the channel at pilot positions, then interpolates to all subcarriers. Simple, fast, but noise is not suppressed.
Key idea: At pilot positions you know both what was sent and what was received, so H_LS = rx_pilot / tx_pilot. One division per pilot. The noise term rides along and degrades performance at low SNR.

The LS problem

H_LS = rx / tx_pilot = (h × tx + noise) / tx = h + noise/tx

The noise term never disappears. At low SNR the noise dominates and the estimate is poor. This is exactly why MMSE and the NN estimator exist.

Interpolation

LS gives you H at pilot positions only. You need H at every data subcarrier. Linear or spline interpolation fills the gaps. More pilots = better interpolation but less space for data.

MATLAB estimate_ls.m
function H_est = estimate_ls(rxGrid, pilotIdx, pilotSyms)
    % LS estimate at pilot positions
    rxPilots = rxGrid(pilotIdx);
    H_pilots = rxPilots ./ pilotSyms;   % one division
    % Interpolate to all subcarriers
    allIdx   = 1:size(rxGrid, 1);
    H_est    = interp1(pilotIdx, H_pilots, allIdx, ...
                        'linear', 'extrap');
end
Interview answer

Q: What is the main limitation of LS channel estimation?
LS estimates the channel by dividing received pilots by known transmitted pilots. The estimate includes the full noise component with no suppression. At low SNR the noise dominates and the channel estimate is poor, leading to significant equalization errors and degraded BLER. MMSE estimation addresses this by incorporating statistical knowledge of the channel and noise to suppress the noise component.

BLOCK 06 / 10
MMSE Channel Estimation
Minimum Mean Square Error estimation blends the noisy LS estimate with statistical knowledge of the channel. It dramatically outperforms LS at low SNR.
Key idea: MMSE applies a weighted correction to LS. At high SNR it trusts the measurement. At low SNR it ignores the noisy measurement and relies on what it knows channels typically look like — the covariance matrix R_hh.

LS vs MMSE comparison

SNR (dB) 5 dB
— true channel — LS estimate — MMSE estimate

MMSE formula

H_MMSE = R_hh (R_hh + σ²I)⁻¹ H_LS

R_hh = channel covariance matrix. σ² = noise variance. As σ² → 0 (high SNR), H_MMSE → H_LS. As σ² → ∞ (low SNR), MMSE relies entirely on R_hh statistics.

Why MMSE still has limits

MMSE requires an accurate R_hh matrix that must be estimated and updated as the channel changes. Matrix inversion is expensive at large antenna counts. If statistics change (user moves environments) R_hh becomes outdated. This opens the door for the NN estimator.

MATLAB estimate_mmse.m
function H_est = estimate_mmse(rxGrid, pilotIdx, pilotSyms, R_hh, SNR_lin)
    % Step 1: LS at pilots
    H_ls     = rxGrid(pilotIdx) ./ pilotSyms;
    % Step 2: MMSE correction
    R_pp     = R_hh(pilotIdx, pilotIdx);
    sigma2   = 1 / SNR_lin;
    W        = R_pp / (R_pp + sigma2 * eye(length(pilotIdx)));
    H_mmse_p = W * H_ls;
    % Step 3: Interpolate to all subcarriers
    H_est    = interp1(pilotIdx, H_mmse_p, 1:size(rxGrid,1), 'linear', 'extrap');
end
Interview answer

Q: How does MMSE improve over LS and why would you want something better?
MMSE blends the noisy LS measurement with channel covariance statistics, suppressing noise especially at low SNR. However it requires accurate R_hh and its inversion is O(n³) — expensive at large antenna counts and degrades when channel statistics change. A neural network can learn these statistics implicitly from data without explicit covariance computation.

BLOCK 07 / 10
Neural Network Estimator
A fully connected network trained on 100,000 synthetic channel realizations learns to denoise and interpolate in a single forward pass — outperforming MMSE at low SNR without needing R_hh.
Key idea: Input = noisy LS estimate at pilot positions. Output = clean channel at all subcarriers. The NN learns the mapping from data — it implicitly learns channel statistics, noise behaviour, and how to weight pilots at every SNR level simultaneously.

Network architecture

Input: 2 × Npilots neurons (real + imaginary parts of H_LS)
Output: 2 × Nsc neurons (real + imaginary of H_est at all subcarriers)
Loss function: MSE = (1/N) Σ |h_true − h_pred|²

Training data generation

For each training example: generate random channel h, compute noisy LS at pilots = input, store true h at all subcarriers = label. Train at random SNR across [-5, 25] dB so the network handles all operating points.

Why NN beats MMSE at low SNR

MMSE treats all pilots equally. The NN learns to distrust noisy pilots and rely more on learned channel structure. It captures nonlinear relationships between pilot observations that linear MMSE cannot. No R_hh computation needed.

MATLAB train_estimator.m
% Build training data
for i = 1:Ntrain
    SNR_dB   = randi([-5 25]);
    sigma2   = 1/(2*10^(SNR_dB/10));
    h_time   = (randn(1,6)+1j*randn(1,6)).*sqrt([.5 .2 .1 .1 .05 .05]);
    h_freq   = fft(h_time, Nsc).';
    noise_p  = sqrt(sigma2)*(randn(1,Np)+1j*randn(1,Np));
    H_ls     = h_freq(pilotIdx) + noise_p.';
    X_train(i,:) = [real(H_ls)', imag(H_ls)'];
    Y_train(i,:) = [real(h_freq)', imag(h_freq)'];
end
% Define and train network
layers = [featureInputLayer(2*Np)
           fullyConnectedLayer(256)  reluLayer
           fullyConnectedLayer(256)  reluLayer
           fullyConnectedLayer(128)  reluLayer
           fullyConnectedLayer(2*Nsc)];
net = trainNetwork(X_train, Y_train, layers, trainingOptions('adam',...));
Interview answer

Q: Why does a neural network outperform MMSE for channel estimation?
MMSE requires an explicit channel covariance matrix, assumes stationary statistics, and uses linear operations. A neural network trained on diverse realizations implicitly learns channel statistics, adapts to varying SNR, and captures nonlinear relationships between pilot observations and the true channel — resulting in better performance especially at low SNR where classical estimators struggle most.

BLOCK 08 / 10
LDPC Decoding
Belief propagation on the sparse Tanner graph iteratively refines soft bit estimates until they converge on a valid codeword. Takes LLRs as input, outputs decoded bits.
Key idea: The decoder passes messages along the sparse graph — variable nodes and check nodes exchange beliefs iteratively. Sparsity means all nodes update in parallel — this is the same parallelism that lets Qualcomm's hardware run at multi-Gbps throughput.

Belief propagation — watch iterations converge

Variable vs Check nodes

Variable nodes (VN) — one per codeword bit. Hold the current LLR belief. Check nodes (CN) — one per parity equation. Enforce the parity constraint. An edge exists if H[row][col] = 1.

Early termination

After each iteration, compute the syndrome H × hard_bits mod 2. If all zeros — valid codeword found, stop early. Real Qualcomm modems use 5-10 iterations with early termination to minimize power and latency.

MATLAB ldpc_decode.m
function bits = ldpc_decode(llrs, bgn, maxIter)
    % MATLAB 5G Toolbox LDPC decoder
    % Positive LLR = likely 0, Negative LLR = likely 1
    bits = nrLDPCDecode(llrs(:), bgn, maxIter);
end

% Full RX decode chain
rxDecoded  = nrLDPCDecode(llrs, bgn, 50);
[~, crcErr] = nrCRCDecode(rxDecoded, '24A');
% crcErr=0 → block OK, crcErr=1 → block error → BLER++
Interview answer

Q: Why does LDPC decoding parallelise better than turbo?
Turbo decoding passes extrinsic information serially between two constituent decoders — a fundamental throughput bottleneck. LDPC belief propagation operates on a sparse graph where all variable nodes and all check nodes update simultaneously each iteration. This maps directly onto silicon — thousands of check node and variable node processors run in parallel, enabling multi-Gbps 5G NR throughput.

BLOCK 09 / 10
BLER Measurement
Block Error Rate is measured by sweeping SNR and counting CRC failures. The SNR at 10% BLER is the standard 3GPP operating point — the metric that shows how much the NN estimator gains over classical methods.
Key idea: Collect at least 100 block errors per SNR point for statistical reliability. At low SNR this is fast (errors come quickly). At high SNR you need many blocks. The 10% BLER crossover tells you how many dB of SNR gain each estimator provides.

True BLER 10%
Errors collected 100

The 10% BLER point

3GPP defines the standard initial transmission target as 10% BLER. HARQ retransmissions handle the remaining failures. The SNR at which each estimator crosses 0.1 BLER is the key number — the gap between LS, MMSE, and NN at this point is your result.

Min errors rule

At BLER = 0.001 you need ~100,000 blocks to collect 100 errors. For a portfolio project, limit the SNR range to where BLER > 0.001. Going lower is diminishing returns. Always report confidence intervals alongside BLER values.

MATLAB bler_loop.m (excerpt)
minErrors = 100;  minBlocks = 1000;
while blockErrors < minErrors || totalBlocks < minBlocks
    % TX chain
    txBits   = randi([0 1], K, 1);
    codeword = nrLDPCEncode(nrCRCEncode(txBits,'24A'), bgn, rate);
    txSig    = ofdm_modulate(mapToGrid(qammod(codeword,M,...)));
    % Channel + RX chain
    rxSig    = awgn_channel(txSig, SNR_dB);
    rxGrid   = ofdm_demodulate(rxSig);
    H_est    = estimate_nn(rxGrid, pilotIdx, pilotSyms, net);
    llrs     = qamdemod(rxGrid./H_est, M, 'OutputType','llr',...);
    [~,err]  = nrCRCDecode(nrLDPCDecode(llrs,bgn,50),'24A');
    totalBlocks = totalBlocks+1;
    if err, blockErrors = blockErrors+1; end
end
BLER = blockErrors / totalBlocks;
Interview answer

Q: How do you ensure BLER results are statistically reliable?
Collect a minimum of 100 block errors per SNR point before accepting the measurement. At low SNR this happens quickly. At high SNR where errors are rare you simulate many more blocks — this is why link-level simulators use early termination on the minimum error count rather than a fixed number of trials.
